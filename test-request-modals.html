Database Tables

The requests are saved in these database tables:

course_requests table:

id, user_id, course_title, category, level, description, status, created_at, updated_at

requested_schools table:

id, user_id, school_name, school_type, level, location, school_email, school_phone, status, created_at, updated_at

API Endpoints

Course Requests:

  • POST /api/course-requests - Create new course request
  • GET /api/course-requests - Get user's course requests
  • GET /api/course-requests/{id} - Get specific course request

School Requests:

  • POST /api/school-requests - Create new school request
  • GET /api/school-requests - Get user's school requests
  • GET /api/school-requests/{id} - Get specific school request

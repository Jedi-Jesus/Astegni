## CSS Grid Template Areas

### Desktop:
```css
grid-template-columns: auto 1fr auto;
grid-template-areas: "left-col content-col right-col";

[Avatar]     [Meta Info Bar           ]  [Price     ]
[Name  ]     [Quote                   ]  [Message   ]
[Actions]    [Subjects                ]  [Connect   ]
             [Details Grid            ]
             [Additional Details      ]
             [Bio                     ]
```

### Tablet:
```css
grid-template-columns: auto 1fr;
grid-template-areas:
    "left-col content-col"
    "left-col right-col";

[Avatar]     [Content                 ]
[Name  ]     [                        ]
[Actions]    [Price + Action Buttons  ]
```

### Mobile:
```css
grid-template-columns: 1fr;
grid-template-areas:
    "left-col"
    "content-col"
    "right-col";

[Avatar + Name + Actions (Horizontal)]
[Content (Full Width)                ]
[Price + Actions (Full Width)        ]
```

## Animation Flow

### Switching from Grid to List:
```
1. User clicks List button
2. .tutor-cards-grid class removed
3. .tutor-cards-list class added
4. CSS Grid recalculates layout (300ms transition)
5. Cards morph from vertical to horizontal
6. Content reflows to 3-column structure
7. Meta info fades in with border
8. Bio truncates with ellipsis
```

## Summary

The list view provides a **horizontal, information-dense layout** optimized for:
- **Quick scanning** of multiple tutors
- **Easy comparison** of key details
- **Efficient use of screen width** on desktop
- **Responsive adaptation** to smaller screens
- **Consistent branding** with grid view
- **Smooth transitions** between views
